Desarrollo de Plataforma Educativa Online para Ingreso a la UNA

Junto con mi hermano Ángel López estamos desarrollando una plataforma web con el objetivo de brindar a los estudiantes herramientas y recursos para prepararse eficazmente para el examen de ingreso a la Facultad Nacional de Asunción, abarcando materias clave como matemáticas, castellano, contabilidad, física, entre otras.

Para este proyecto se utilizó HTML, CSS, JavaScript, PHP y mySQL, con un diseño responsivo, lo que nos permite crear una aplicación web dinámica e interactiva. El desarrollo tiene las siguientes secciones: Home (interfaz principal), Dashboard (panel de control para seguimiento del progreso), Login/Registro (sistema de autenticación), Materias (organización de contenidos) y Lecciones (visualización de lecciones con ejercicios interactivos). Nos enfocamos en crear una experiencia de usuario intuitiva y accesible.

Desarrollo del Home

La página de inicio (Home) actúa como la principal puerta de entrada para los usuarios a la plataforma. Su diseño se centró en facilitar la navegación y el acceso rápido a los contenidos relevantes.

Habilidades desarrolladas:

  • Diseño UX: Priorización de la usabilidad e interfaz intuitiva.
  • Integración BD: Conexión y consulta a MySQL con PHP para contenido dinámico.
  • Desarrollo API: Comprensión de la comunicación front-end/back-end.
  • Consultas SQL: Manejo eficiente de consultas para obtener y mostrar datos.

Desarrollo del Registro - Login

La página de Login permite a los usuarios registrados acceder a la plataforma de forma segura. Se implementó un sistema de autenticación que verifica las credenciales ingresadas (usuario y contraseña) contra la base de datos.

Habilidades desarrolladas:

  • Autenticación de usuarios: Implementación de un sistema de login funcional, incluyendo validación de campos y verificación de credenciales.
  • Seguridad: Aprendí sobre la importancia del hashing de contraseñas para proteger la información sensible de los usuarios.
  • Sesiones: Adquirí conocimientos sobre el manejo de sesiones en PHP para mantener al usuario autenticado durante su navegación en la plataforma.
  • Experiencia de usuario (UX): Se diseñó un formulario de login intuitivo y fácil de usar, con etiquetas claras.

Desarrollo de las lecciones

La sección de Lecciones presenta el contenido educativo organizado por materia. Al seleccionar una lección, se realiza una consulta a la base de datos para obtener la información específica de esa lección, incluyendo título, descripción y contenido. Además, se muestra información adicional como la cantidad de vistas y los votos recibidos por cada lección.

Habilidades desarrolladas:

  • Consultas dinámicas a la base de datos: Tiene consultas SQL dinámicas para obtener la información específica de cada lección según la selección del usuario, optimizando el rendimiento y evitando la necesidad de crear páginas estáticas para cada lección.
  • Organización y presentación de contenido: Aprendí a estructurar y presentar la información de las lecciones de forma clara y organizada, incluyendo título, descripción y contenido principal.
  • Manejo de datos adicionales: Se implementó la lógica para mostrar información adicional como la cantidad de vistas y los votos, lo que requiere realizar consultas adicionales y procesar los resultados.
  • Optimización de consultas: Consideramos la eficiencia de las consultas a la base de datos para asegurar un tiempo de carga rápido, especialmente al manejar un gran volumen de lecciones.
  • Relaciones entre tablas: Comprendí la importancia de las relaciones entre tablas en la base de datos para vincular las lecciones con otra información, como las materias a las que pertenecen, los usuarios que las han visto o votado, etc.
  • Presentación de datos al usuario (Front-end): Mostramos la información obtenida de la base de datos de forma atractiva y fácil de entender para el usuario, utilizando HTML, CSS y JavaScript para mejorar la experiencia de usuario.

Desarrollo del Juego

La sección de Juego ofrece actividades interactivas para reforzar el aprendizaje. Los juegos se generan dinámicamente con JavaScript, aunque también se incluyen algunos juegos estáticos. Cada juego presenta un enunciado, opciones múltiples y un temporizador. El usuario debe seleccionar la opción correcta. Al finalizar, se guarda el progreso del usuario en la base de datos, incluyendo el tiempo empleado, las respuestas correctas e incorrectas.

Habilidades desarrolladas:

  • Generación dinámica de contenido (JavaScript): Implementé la lógica con JavaScript para generar juegos dinámicos, lo que me permitió crear una variedad de actividades sin necesidad de crear código HTML estático para cada juego.
  • Manejo del DOM (JavaScript): Adquirí experiencia en la manipulación del DOM (Document Object Model) para mostrar el enunciado, las opciones, el temporizador y para gestionar las interacciones del usuario con el juego.
  • Temporizadores (JavaScript): Se agregó un temporizador con JavaScript para controlar el tiempo disponible para cada juego y para gestionar el final del juego.
  • Comunicación con el servidor (Fetch): Utilicé Fetch para enviar los datos del juego (tiempo, respuestas correctas/incorrectas) al servidor y guardarlos en la base de datos.
  • Almacenamiento de datos en la base de datos: Aprendí a diseñar la estructura de la base de datos para almacenar la información del progreso del usuario en los juegos, incluyendo el tiempo, las respuestas correctas e incorrectas.
  • Lógica de juego y validación de respuestas (JavaScript): Implementé la lógica para validar las respuestas del usuario y para calcular la puntuación final.
  • Experiencia de usuario (UX) en juegos: Consideré la experiencia del usuario al diseñar la interfaz del juego, asegurando que fuera intuitiva, atractiva y fácil de usar.

Desarrollo del Dashboard

El Dashboard actúa como un panel de control administrativo, permitiendo la gestión de contenido y usuarios. Desde esta sección, los administradores pueden agregar y editar lecciones, así como visualizar la lista de usuarios registrados en la plataforma.

Habilidades desarrolladas:

  • CRUD (Crear, Leer, Actualizar, Borrar): Implementé las operaciones CRUD para la gestión de lecciones, lo que me permitió crear un sistema completo para administrar el contenido educativo.
  • Formularios de administración: Desarrollé formularios para agregar y editar lecciones, incluyendo la validación de los datos ingresados para asegurar la integridad de la información.
  • Listado y visualización de usuarios: Se desarrolló una interfaz para mostrar la lista de usuarios registrados, lo que permite a los administradores tener una visión general de la base de usuarios.
  • Paginación: Implementé paginación para mejorar el rendimiento y la usabilidad al mostrar grandes cantidades de datos.
  • Interfaz de usuario (UI) para administración: Diseñé una interfaz de usuario intuitiva y fácil de usar para las funciones de administración, lo que facilita la gestión del contenido y los usuarios.

Desarrollo de la Tabla de Progreso

La Tabla de Progreso, integrada en el Dashboard, proporciona una visión general del rendimiento de los usuarios en las diferentes materias y ejercicios. Muestra información clave como las materias cursadas, la cantidad de ejercicios realizados, el número de respuestas correctas e incorrectas, permitiendo a los administradores y/o usuarios monitorear el avance y la comprensión de los contenidos.

Habilidades desarrolladas:

  • Recopilación y carga de datos: Implementé la lógica para recopilar y agregar datos de diferentes fuentes (ejercicios, usuarios, materias) para generar la información que se muestra en la tabla de progreso. Esto implica realizar consultas a múltiples tablas en la base de datos y procesar los resultados.
  • Visualización de datos en tablas: Implementé la visualización de los datos en una tabla clara y organizada, utilizando HTML y CSS. Consideré la usabilidad y la legibilidad al diseñar la tabla.
  • Presentación de datos al usuario (Front-end): Aprendí a mostrar la información de manera efectiva para que sea fácilmente interpretable por los usuarios.
